# COZY-VUE PROJECT ARCHITECTURE & RULES

## PROJECT OVERVIEW
- **Type**: Vue 3 Social Media Application with Firebase Integration
- **Mobile Ready**: Capacitor Android support
- **State Management**: Composition API with centralized stores
- **Authentication**: Firebase Auth with social login support

---

## ARCHITECTURE

### src/composables/
- **useAuth.js**: Quản lý xác thực người dùng (login, signup, forgot password, logout, social login)
- **useSettings.js**: Hợp nhất quản lý theme, ngôn ngữ, storage (localStorage operations)
- **useValidation.js**: Kiểm tra và xác thực dữ liệu đầu vào của forms

### src/components/
- **HomeFeatures.vue**: Hiển thị các tính năng chính 
- **LoginForm.vue**: Form đăng nhập/đăng ký với tab switching
- **LoginSocial.vue**: Đăng nhập bằng mạng xã hội + QR code placeholder
- **NavMid.vue**: Thanh điều hướng giữa với 5 navigation items
- **NavRight.vue**: Thanh điều hướng phải (theme switcher, language, logout)

### src/views/
- **Home.vue**: Giao diện trang chủ với layout 3 cột
- **Login.vue**: Giao diện trang đăng nhập với layout 3 cột

### src/router/
- **index.js**: Cấu hình router chính với route guards (requiresAuth, requiresGuest)

### src/stores/
- **authStore.js**: Quản lý trạng thái xác thực với Firebase onAuthStateChanged

### src/assets/
- **main.css**: CSS variables và responsive grid layout (3-column design)

### Root Files:
- **App.vue**: Root component với NavRight và router-view
- **main.js**: App initialization với router và CSS
- **firebase.js**: Firebase configuration và auth helpers
- **index.html**: HTML entry point
- **capacitor.config.json**: Capacitor mobile app configuration

## CODE RULES

### 1. Single Responsibility Principle
- **Mỗi composable chỉ thực hiện 1 chức năng domain cụ thể**
- **Mỗi component chỉ đảm nhiệm 1 UI block**
- **Hợp nhất các composables có chức năng liên quan (Settings = Theme + Language + Storage)**

### 2. Composables Structure
```javascript
// Template cho composables
export function useFeatureName() {
  // 1. Imports và dependencies
  // 2. Global state initialization (nếu cần)
  // 3. Reactive state và computed
  // 4. Methods/Functions
  // 5. Lifecycle hooks (onMounted nếu cần)
  // 6. Return object với grouped methods
}
```

### 3. Error Handling Pattern
- **Centralized error messages trong useAuth với i18n support**
- **withErrorHandling wrapper cho tất cả async operations**
- **Consistent error display qua global state**
- **Loading states được manage centrally**

### 4. State Management Architecture
- **Global state: Firebase auth state qua authStore**
- **Settings state: localStorage persistence qua useSettings**
- **Component state: local reactive state trong composables**
- **No direct localStorage access từ components**

### 5. Component Communication
- **Props down, events up cho parent-child**
- **Composables cho cross-component logic sharing**
- **Router để communicate qua URL state**
- **Global stores chỉ cho authentication state**

## LOGIC RULES

### 1. Authentication Flow
```
Login/Signup → useAuth → Firebase Auth → authStore → Router Navigation
Social Login → Firebase Social Providers → authStore → Auto redirect
Logout → authStore.logout → Firebase signOut → Router redirect
Route Guards → authStore.isLoggedIn → Redirect logic
```

### 2. Settings Flow
```
Theme Change → useSettings.changeThemeColor → CSS Variables + localStorage
Language Toggle → useSettings.toggleLanguage → Reactive translations + localStorage
Storage Operations → useSettings generic methods → localStorage with error handling
```

### 3. Validation Flow
```
Form Submit → useValidation methods → Error return → useAuth.handleError → Display
Email/Password validation → Consistent error keys → i18n error messages
```

### 4. Data Persistence Flow
```
User Preferences → useSettings → localStorage → Page refresh → Auto restore
Auth State → Firebase → authStore → Browser refresh → Firebase SDK restore
Form Remember → Encrypted localStorage → Page load → Auto populate
```

### 5. Mobile App Flow
```
Vue App → Capacitor → Android WebView → Native device features
Build process → npm run build → Capacitor sync → Android Studio
```

## OPERATION RULES

### 1. Development Workflow
- **Không thay đổi giao diện khi chưa được yêu cầu**
- **Không thêm/bớt chức năng khi chưa được yêu cầu**
- **Chỉ giữ lại các logic cần thiết**
- **Loại bỏ logic/code trùng lặp, dư thừa (optimization allowed)**
- **Không xóa comments hiện có**
- **Maintain 3-column responsive layout pattern**

### 2. Code Quality Standards
- **Vue 3 Composition API pattern tuân thủ strict**
- **TypeScript ready structure (có thể upgrade)**
- **Consistent naming: camelCase cho tất cả**
- **Error boundaries và graceful degradation**
- **Mobile-first responsive design**

### 3. Performance Rules
- **Component lazy loading cho large views**
- **Composable method debouncing cho user inputs**
- **Cleanup timers/listeners trong onUnmounted**
- **Optimize reactive dependencies**
- **Firebase connection pooling**

### 4. Security Requirements
- **Password encryption trước localStorage (simple Base64)**
- **Input validation qua useValidation**
- **Firebase security rules compliance**
- **No sensitive data trong client-side storage**
- **HTTPS only cho production**

### 5. Mobile Development
- **Capacitor Android build support**
- **Touch-friendly UI components**
- **Responsive breakpoints trong CSS**
- **Native device API ready structure**
